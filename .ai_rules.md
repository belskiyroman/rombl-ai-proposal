# Mandatory Developer Rules for This Project

You MUST follow these strict guidelines for *every* interaction and code generation task within this repository.

## 1. Workflow Methodology: Strict TDD ALWAYS
Before writing any implementation code, you **MUST** create a failing test that defines the required functionality.
* **RED:** Write a failing test.
* **GREEN:** Write minimal implementation to make it pass.
* **REFACTOR:** Clean up, ensure all tests still pass.

## 2. Technical Output Constraints
* **Language:** All generated code and test files **MUST** be in TypeScript.
* **Context:** Use `AGENTS.md` as the absolute source of truth for agent roles, prompts, schemas, and orchestration logic. Do not deviate from it.

## 3. Communication Style
* **Autonomy:** Do not ask for permission to execute a task. Work iteratively and provide results. If you need clarification on a spec, ask, otherwise proceed with TDD.

## 4. Git Commits
* **Always start a new task with:** `git checkout -b <prefix>/<branch-name>`
* **Commit Message:** Use `Conventional Commits` format.
* **Scope:** Use `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`.
* **Body:** Use imperative mood. Describe *what* changed and *why*.
* **Always push after each step:** `git push -u origin <prefix>/<branch-name>`
